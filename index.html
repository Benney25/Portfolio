Jeremy’s IT Lab Day 24 Lab – Floating Static Routes



1. Check the routing tables of R1 and R2.

  1. Which dynamic routing protocol is Enterprise A using?

``

Code translations: O = OSPF, S = Static, * = Candidate Default

0.0.0.0/0 via x.x.x.x = default gateway


R1#show ip route

O 10.0.2.0/24 [110/2] via 10.0.0.2

S* 0.0.0.0/0 [1/0] via 203.0.113.9


R2#show ip route

O 10.0.1.0/24 [110/2] via 10.0.0.1

S* 0.0.0.0/0 [1/0] via 203.0.113.13

``


This means that Enterprise A is using OSPF Internally, and Static Routing to set their default gateway.

Answer from lab walkthrough:

```

R1(config)#ip route 10.0.2.0 255.255.255.0 203.0.113.1 111

```

After executing this command, the route appears in the routing table for R1. Statically configuring the route from SRV1 to PC1 on R2 with the same process has similar results. Note that during this time the original route has remained down.

A ping from PC1 to SRV1 fails ¾ times due to ARP Resolution, but final packet succeeds. With 3 Routers in transit to SRV1, each of 3 successive pings were dropped as the ARP Resolution process was started.

Key takeaways:

Do not enter interface-configuration mode to configure floating static routes

A floating static route is a static route. You have to configure it as a static route or it will not work.
